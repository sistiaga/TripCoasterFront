Actúa como un arquitecto senior de frontend especializado en Angular y Angular Material.

Contexto del proyecto:
- Aplicación Angular con múltiples módulos y componentes.
- Actualmente usa CSS/SCSS personalizado por componente y estilos globales dispersos.
- Hay múltiples formularios, tablas y vistas CRUD.
- Se usan Reactive Forms.
- El objetivo es unificar y estandarizar todo el diseño usando Angular Material.

Objetivos principales:
1. Migrar la UI existente a Angular Material sin romper la lógica de negocio.
2. Definir una arquitectura de estilos clara, limpia y escalable.
3. Eliminar CSS repetido y anti-patrones (clases genéricas, ::ng-deep, overrides innecesarios).
4. Centralizar el diseño visual mediante un sistema de temas de Angular Material.
5. Asegurar consistencia visual en toda la aplicación.

Requisitos técnicos:
- Usar Angular Material como base visual (inputs, botones, tablas, diálogos).
- Usar SCSS.
- Implementar un theme global (primary, accent, warn).
- Separar correctamente:
  - estilos globales
  - tema de Angular Material
  - estilos específicos de componentes
- Mantener ViewEncapsulation por defecto.
- Evitar CSS global innecesario.
- Preparar el proyecto para soporte de temas (ej: claro/oscuro).

Qué necesito que hagas:
1. Analizar la estructura actual de estilos (conceptualmente).
2. Proponer una arquitectura de carpetas para estilos (theme, typography, layout, helpers).
3. Definir qué estilos deben ser globales y cuáles deben ir en cada componente.
4. Proponer un sistema de estandarización para:
   - Formularios
   - Botones
   - Tablas
   - Mensajes de error
   - Layouts comunes (páginas, cards, modales)
5. Indicar patrones recomendados para:
   - Formularios reutilizables con Angular Material
   - Uso correcto de mat-form-field
   - Manejo de validaciones y errores
6. Dar ejemplos concretos de:
   - Theme SCSS de Angular Material
   - Un formulario típico migrado a Angular Material
   - Un componente bien estructurado (HTML + SCSS)
7. Detectar y corregir posibles problemas comunes:
   - Overriding incorrecto de estilos Material
   - Uso excesivo de CSS global
   - Inconsistencias visuales entre vistas

Formato de la respuesta:
- Explicaciones claras y prácticas
- Ejemplos de código cuando sea necesario
- Reglas y convenciones explícitas
- Recomendaciones basadas en buenas prácticas reales de Angular

No simplifiques en exceso.
Prioriza mantenibilidad, escalabilidad y consistencia visual.
